# Always-available streams

The server can be configured to serve continuous streams (always-available streams), that are generated by concatenating "online" streams (sent by an online publisher) with an "offline" video, that is played on repeat when the publisher is offline. Concatenation happens by using already-encoded packets, without decoding or re-encoding anything, avoiding any additional resource consumption.

This feature can be enabled by toggling the `alwaysAvailable` flag:

```yml
paths:
  mypath:
    alwaysAvailable: true
```
